> *DRAFT — NOT FOR DISTRIBUTION*

**Information Security Policy**

Version 0.8 — DRAFT \| February 2024

This document is a draft and has not been approved for publication. Do not distribute outside the Information Security team.

1\. Introduction

This policy establishes the information security requirements for all DWP systems, data, and personnel. It replaces the Information Security Policy v4.1 (June 2022).

> *\[REVIEWERS NOTE: Section 2 needs legal review before publication. The DPIA requirements may conflict with the updated UK GDPR guidance from ICO published in January 2024.\]*

2\. Classification

All information must be classified according to the Government Security Classifications policy:

- OFFICIAL: The majority of government information

- SECRET: Very sensitive information requiring enhanced protection

- TOP SECRET: The most sensitive information

Within OFFICIAL, the marking OFFICIAL-SENSITIVE may be applied where there is a need to enforce need-to-know.

3\. Access Control

Access to departmental systems must follow the principle of least privilege. Users should only have access to the systems and data they need for their current role.

3.1 Passwords

Minimum length: 12 characters

Complexity: Must include uppercase, lowercase, numbers, and special characters

Expiry: Every 90 days

> *\[TODO: Update password section to reflect NCSC guidance. NCSC now recommends against forced password expiry. Need to align with NCSC Cyber Essentials requirements.\]*

3.2 Multi-factor authentication

MFA is required for all remote access and for access to systems handling OFFICIAL-SENSITIVE or above.

4\. Device Security

4.1 Department-issued devices

All laptops and mobile devices must be encrypted using BitLocker (Windows) or FileVault (Mac).

Automatic updates must be enabled. Devices must not be used if updates are more than 14 days overdue.

4.2 Personal devices

Personal devices must not be used to access departmental systems or data unless through an approved BYOD solution. Currently, no BYOD solution is approved.

> *\[PLACEHOLDER: Section 5 — Cloud Security — to be drafted by Cloud Architecture team. Target: March 2024.\]*

6\. Incident Response

All security incidents must be reported to the Security Operations Centre (SOC) immediately.

SOC contact: soc@dwp.gov.uk / 0800 XXX XXXX (24/7)

Severity classification:

P1 — Active compromise or data breach affecting live services

P2 — Suspected compromise or vulnerability being actively exploited

P3 — Security policy violation or minor vulnerability

P4 — Advisory or informational

7\. Training

All staff must complete Responsible for Information (RfI) training annually. Staff in roles with elevated access must complete additional security training as specified by their role requirements.

> *\[END OF DRAFT — Sections 5, 8, 9, 10 still to be written\]*
